自定义View概述

1.关于坐标系

  • Android系统手机屏幕的左上角为坐标系的原点，y轴方向与笛卡尔坐标系的y轴方向想反,x轴方向不变。
  • 角度:从x轴开始,顺时针旋转为正
    • 角度与弧度关系: 180度 = π弧度

1.2获取控件的坐标和触摸点的坐标

  • 获取控件的坐标

    • 通过提供的API(如getLeft , getTop, getBottom, getRight)可以获得控件在parent中的相对位置.

    • 获取view在屏幕中的位置:需要在onLayout完后获取,通常在监听布局完成后获取

  • 获取触摸点的坐标

    • MotionEvent的方法getX(),getY()获取触摸点相对于其所在组件坐标系的坐标
    • getRawX()和getRawY()获取触摸点相对于屏幕默认坐标系的坐标

3.2 ViewConfiguration类

  • 说明:提供了一些自定义控件用到的标准常量，比如尺寸大小，滑动距离，敏感度等等
  • 获取实例:
    • ViewConfiguration viewConfiguration=ViewConfiguration.get(context);

3.3 GestureDetector类

  • 说明:手势监听(单击,滑动,长按,双击等)处理的工具类(内部有一个OnGestureListener接口),当然我们也可以在onTouchEvent()中自己处理手势

  • 使用步骤:

    1. 创建一个类实现OnGestureListener
      • onDown: 触摸屏幕时调用该方法
      • onFling: 手指在屏幕拖动时调用该方法
      • onLongPress: 长按屏幕时调用该方法
      • onScroll: 滚动时调用该方法
      • onShowPress: 手指在屏幕上按下,且未移动和松开时调用该方法
      • onSingleTapUp: 轻击屏幕时调用该方法
    2. new 一个GestureDetector对象,构造方法中传一个context和第一步创建的类

    3. 将Touch事件交给GestureDetector处理

3.4 VelocityTracker

  • 说明:用于跟踪触摸屏事件（比如，Flinging及其他Gestures手势事件等）的速率(包括水平和竖直方向)

  • 使用步骤:

    1. 开始速度追踪

    2. 获取追踪到的速度

    3. 解除速度追踪: 在不需要的时候重置并回收内存

3.5 Scroller

  • 说明:
    • 用于跟踪View的滑动(view的scrollTo/scrollBy滑动是在瞬间完成,没有过度效果)的轨迹，实现过渡效果的滑动.此类不会移动控件，只移动View的内容
    • Scroller本身无法让view滑动,需要和view的computeScroll方法结合使用

      View中的scrollTO和scrollBy方法:用于滑动View中的内容，而不是把某个View的位置进行改变
      改变view在屏幕中的位置offsetLeftAndRight(int offset)用于左右移动方法或offsetTopAndBottom(int offset)用于上下移动

  • 使用步骤:

    1. 传入context,创建Scroller对象

    2. 复写view的computeScroll方法

    3. 调用Scroller的startScroll方法,在调用invalidate方法进行重绘

      //封装缓慢滑动到指定位置方法
private void smoothScrollTo(int desX , int desY){
    int scrollX = getScrollX();
    int deltaX = desX - scrollX;
    //指定1s内滑动到desX
    mScroller.startScroll(scrollX,0,deltaX,0,1000);
}

2.3 获取电话的状态

  • 来电:

    • 一般在服务中进行

    • 思路:

      • 调用系统服务获取TelephonyManager
      • 调用listen方法
      • 设置监听事件,在PhoneStateListener中复写onCallStateChanged方法
      • 获取三种状态

    • 挂断电话:(AIDL远程服务)

      • 查看源码(aidl文件sourceInsight无法查看)

        • TelephonyManager–>getITelephony(); —android.jar包中没有该方法
        • 找到aidl文件的路径:com.android.internal.telephony.ITelephony
        • 文件夹路径: android4.0\frameworks\base\telephony\java\com\android\internal\telephony\ITelephony.aidl
        • 网站查询:www.grepcode.com–>TelephonyManager–>getITelephony()
        • ITelephony接口下有endCall方法
        • 查找:NeighboringCellInfo.aidl
          • android4.0\frameworks\base\telephony\java\android\telephony\NeighboringCellInfo.aidl

      • 思路:

        • ITelephony接口中才有endCall的方法
        • ITelephony接口类型实例的获取可调用getITelephony()方法
        • 而ITelephony.aidl中提供了获取ITelephony的方法
          ITelephony telephony = ITelephony.Stub.asInterface(ServiceManager.getService(Context.TELEPHONY_SERVICE));
        • ServiceManager未暴露出来,故需使用反射的手段获取其中的getService方法

  • 去电:

    • 通过广播接受者获取
      • 思路:
        • 在onReceive方法中调用getResultData方法来获取数据
