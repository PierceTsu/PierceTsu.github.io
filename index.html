Paint、Canvas和Path类的使用介绍

一、Paint

  • Paint类为Android中的画笔类
  • Paint集成了所有“画”的属性

1.1 画笔的初始化

  • 一般在构造方法中进行初始化,切记不要在onDraw方法中初始化,将会多次创建对象

  • 各种setter方法来设置各种不同的属性

    比如setColor()设置画笔颜色，setStrokeWidth()设置描边线条，setStyle()设置画笔的样式：

1.2 相应API

  • setStyle(Style style);

    画笔样式分三种：

    1. Paint.Style.STROKE：描边
    2. Paint.Style.FILL_AND_STROKE：描边并填充
    3. Paint.Style.FILL：填充
  • set(Paint src);

    顾名思义为当前画笔设置一个画笔，说白了就是把另一个画笔的属性设置Copy给我们的画笔

1.2.1 颜色相关操作

  • setColorFilter(ColorFilter filter);

    设置颜色过滤,传入一个ColorFilter参数同样也会返回一个ColorFilter实例,其子类有三个

    • ColorMatrixColorFilter:色彩矩阵颜色过滤器,该构造方法中需要传入ColoMatrix对象
      • ColoMatrix对象定义了一个4x5的float[]类型的矩阵：
    

* 这个矩阵不同的位置表示的RGBA值，其范围在0.0F至2.0F之间，1为保持原图的RGB值。每一行的第五列数字表示偏移值
    • LightingColorFilter:光照颜色过滤,构造方法:LightingColorFilter (int mul, int add),这两个值都是16进制的色彩值0xAARRGGBB
      • mul全称是colorMultiply意为色彩倍增
      • add全称是colorAdd意为色彩添加
    • PorterDuffColorFilter:混合模式,构造方法:PorterDuffColorFilter(int color,PorterDuff.Mode mode)
      • color:表示16进制的颜色值
      • mode:表示PorterDuff内部类Mode中的一个常量值，这个值表示混合模式(该模式不仅仅是应用于图像色彩混合，还应用于图形混合)

  • setXfermode(Xfermode xfermode);

    设置图片混合的过渡模式,与setColorFilter类似,传入一个Xfermode参数同样也会返回一个Xfermode实例,其子类有三个

    • AvoidXfermode,这个API因为不支持硬件加速在API 16已经过时了,如需使用在AndroidManifest.xml文件中设置application节点下的android:hardwareAccelerated属性为false来关闭硬件加速
    • PixelXorXfermode,与AvoidXfermode一样也在API 16过时了
    • PorterDuffXfermode,构造方法:PorterDuffXfermode(PorterDuff.Mode mode)
1.2.1.1 颜色混合模式
  • 不同模式时的混合效果如下:
  • 说明:
    • Sa全称为Source alpha表示源图的Alpha通道；
    • Sc全称为Source color表示源图的颜色；

    • Dc全称为Destination color表示目标图的颜色

    • XOR: 在均为不透明情况下可以使用src去切除与dst相交的地方,使两张图片保留空隙

1.2.2 字体相关操作

  • 获取文字的相关属性
    • measureText,有三个重载方法,返回文字的宽度(注意，请在设置完文本各项参数后调用)
    • getFontMetrics方法,返回一个FontMetrics对象

      FontMetrics字体测量类:Paint的一个内部类,定义了五个参数,可以在绘制文本前获取,与绘制文本的内容无关,仅与绘制文本Paint的size和typeface有关

      • top
      • ascent:baseline以上的距离,为负
      • descent:baseline以下的距离,为正
      • bottom,
      • leading:上一行字符的descent到该行字符的ascent之间的距离
    • descent()方法:文本绘制中返回baseline以下的距离,为正
    • ascent()方法:文本绘制中返回baseline以上的距离,为负
    • getFontSpacing()方法:返回字符行间距
  • breakText,有三个重载方法,设置一个最大宽度在不超过这个宽度的范围内返回实际测量值否则停止测量
  • setUnderlineText(boolean underlineText)设置下划线

  • setTypeface(Typeface typeface)设置字体类型(5种)

    Typeface这个类中给我们提供了多个方法去个性化我们的字体

同样TextView中可以获取到TextPaint,设置字体调用setTypeface方法

  • setTextSkewX(float skewX)这个方法可以设置文本在水平方向上的倾斜,值为负右倾值为正左倾，默认值为0
  • setTextScaleX (float scaleX)将文本沿X轴水平缩放，默认值为1，当值大于1会沿X轴水平放大文本，当值小于1会沿X轴水平缩放文本,不仅放大了文本宽度同时还拉伸了字符
  • setTextLocale (Locale locale)设置地理位置
  • setTextAlign (Paint.Align align)设置在baseline绘制文本究竟该从何处开始，可供选的方式有三种：CENTER(从文本的中点开始向两端绘制),LEFT和RIGHT
  • setStrikeThruText (boolean strikeThruText)文本删除线
  • setFakeBoldText (boolean fakeBoldText)设置文本仿粗体
  • setDither(boolean dither)这玩意用来设置我们在绘制图像时的抗抖动，也称为递色
  • setMaskFilter(MaskFilter maskfilter)
    MaskFilter有两个子类
    • BlurMaskFilter:模糊遮罩滤镜
    • EmbossMaskFilter:浮雕遮罩滤镜
  • setPathEffect(PathEffect effect)设置路径效果
    PathEffect有6个子类,具体可查看源码
  • setStrokeCap(Paint.Cap cap)设置我们画笔的笔触风格
  • setStrokeJoin(Paint.Join join)设置结合处的形态
  • setShadowLayer(float radius, float dx, float dy, int shadowColor)该方法为我们绘制的图形添加一个阴影层效果,不支持硬件加速
  • setShader(Shader shader)给图像着色
    Shader,有5个子类
    • BitmapShader:载入一张图片来给图像着色
    • LinearGradient:线性渐变
    • SweepGradient:梯度(扫描)渐变
    • RadialGradient:径向渐变
    • ComposeShader:组合Shader

1.2.3 TextPaint

  • 能够在绘制时为文本添加一些额外的信息，这些信息包括:baselineShift,bgColor,density,drawableState,linkColor
  • StaticLayout结合TextPaint实现换行

二、Canvas

  • Canvas类为Android中的画布类
  • Canvas下的各类drawXXX方法绘制各种不同的东西

2.1 Canvas介绍

  • 提供了各种draw方法

    比如绘制一个圆drawCircle()，绘制一个圆弧drawArc()，绘制一张位图drawBitmap()等等

  • 在View的onDraw方法会传这个参数过来,该对象拥有一些绘制的上下文关联

2.2 drawXXX为主的绘制方法

2.2.1 绘制文字

  • 指定文本基线位置
    • drawText();
  • 分别指定每个文字的位置(弃用)
    • drawPostText();
  • 根据路径绘制文字
    • drawTextOnPath();

2.2.2 clipXXX为主的裁剪方法(设置画布的显示区域)

  • clipPath(Path path);,以Path的方式创建更多不规则的裁剪区域
  • clipRect方法,有7个重载方法

    Rect和RectF类

    • intersect方法,取相交的区域
    • union方法,取的是相交区域最远的左上端点作为新区域的左上端点，而取最远的右下端点作为新区域的右下端点

2.2.3 以scale、skew、translate和rotate组成的Canvas变换方法

  • 平移和旋转: translate和rotate()
    • 说明:
      • 通过画布的操作可以简化绘制坐标计算
      • 画布的平移旋转同样也会影响画布的自身坐标
  • drawBitmapMesh(Bitmap bitmap, int meshWidth, int meshHeight, float[] verts, int vertOffset, int[] colors, int colorOffset, Paint paint);按照meshWidth和meshHeight这两个参数的值将我们的图片划分成一定数量的网格,通过移动这些交点来改变图像

2.2.4 以saveXXX和restoreXXX构成的画布锁定和还原

  • 释放和锁定画布:restore();和canvas.save();

2.2.5 set方法

  • setBitmap();为Canvas指定一个Bitmap对象,需要一个Bitmap对象来保存像素

三、Path

  • 封装了由直线和曲线(二次，三次贝塞尔曲线)构成的几何路径。你能用Canvas中的drawPath来把这条路径画出来(同样支持Paint的不同绘制模式)，也可以用于剪裁画布和根据路径绘制文字。我们有时会用Path来描述一个图像的轮廓，所以也会称为轮廓线(轮廓线仅是Path的一种使用方法，两者并不等价)

3.1 常用方法

作用 相关方法 备注
移动起点 moveTo 移动下一次操作的起点位置
设置终点 setLastPoint 重置当前path中最后一个点位置，如果在绘制之前调用，效果和moveTo相同
连接直线 lineTo 添加上一个点到当前点之间的直线到Path,可以考虑多次调用lineTo方法来绘制更复杂的图形
闭合路径 close 连接第一个点连接到最后一个点，形成一个闭合区域
添加内容 addRect, addRoundRect, addOval, addCircle, addPath, addArc, arcTo 添加(矩形， 圆角矩形， 椭圆， 圆， 路径， 圆弧) 到当前Path (注意addArc和arcTo的区别)
是否为空 isEmpty 判断Path是否为空
是否为矩形 isRect 判断path是否是一个矩形
替换路径 set 用新的路径替换到当前路径所有内容
偏移路径 offset 对当前路径之前的操作进行偏移(不会影响之后的操作)
贝塞尔曲线 quadTo, cubicTo 分别为二次和三次贝塞尔曲线的方法
rXxx方法 rMoveTo, rLineTo, rQuadTo, rCubicTo 不带r的方法是基于原点的坐标系(偏移量)， rXxx方法是基于当前点坐标系(偏移量)

提示方法 incReserve 提示Path还有多少个点等待加入(这个方法貌似会让Path优化存储结构)
布尔操作(API19) op 对两个Path进行布尔运算(即取交集、并集等操作)

自定义View概述

一、关于坐标系

  • Android系统手机屏幕的左上角为坐标系的原点，y轴方向与笛卡尔坐标系的y轴方向相反,x轴方向不变。
  • 角度:从x轴开始,顺时针旋转为正
    • 角度与弧度关系: 180度 = π弧度

1.获取控件的坐标和触摸点的坐标

  • 获取控件的坐标

    • 通过提供的API(如getLeft , getTop, getBottom, getRight)可以获得控件在parent中的相对位置.

    • 获取view在屏幕中的位置:需要在onLayout完后获取,通常在监听布局完成后获取

  • 获取触摸点的坐标

    • MotionEvent的方法getX(),getY()获取触摸点相对于其所在组件坐标系的坐标
    • getRawX()和getRawY()获取触摸点相对于屏幕默认坐标系的坐标

3.3 GestureDetector类

  • 说明:手势监听(单击,滑动,长按,双击等)处理的工具类(内部有一个OnGestureListener接口),当然我们也可以在onTouchEvent()中自己处理手势

  • 使用步骤:

    1. 创建一个类实现OnGestureListener
      • onDown: 触摸屏幕时调用该方法
      • onFling: 手指在屏幕拖动时调用该方法
      • onLongPress: 长按屏幕时调用该方法
      • onScroll: 滚动时调用该方法
      • onShowPress: 手指在屏幕上按下,且未移动和松开时调用该方法
      • onSingleTapUp: 轻击屏幕时调用该方法
    2. new 一个GestureDetector对象,构造方法中传一个context和第一步创建的类

    3. 将Touch事件交给GestureDetector处理

3.4 VelocityTracker

  • 说明:用于跟踪触摸屏事件（比如，Flinging及其他Gestures手势事件等）的速率(包括水平和竖直方向)

  • 使用步骤:

    1. 开始速度追踪

    2. 获取追踪到的速度

    3. 解除速度追踪: 在不需要的时候重置并回收内存

3.5 Scroller

  • 说明:
    • 用于跟踪View的滑动(view的scrollTo/scrollBy滑动是在瞬间完成,没有过度效果)的轨迹，实现过渡效果的滑动.此类不会移动控件，只移动View的内容
    • Scroller本身无法让view滑动,需要和view的computeScroll方法结合使用

      View中的scrollTO和scrollBy方法:用于滑动View中的内容，而不是把某个View的位置进行改变
      改变view在屏幕中的位置offsetLeftAndRight(int offset)用于左右移动方法或offsetTopAndBottom(int offset)用于上下移动

  • 使用步骤:

    1. 传入context,创建Scroller对象

    2. 复写view的computeScroll方法

    3. 调用Scroller的startScroll方法,在调用invalidate方法进行重绘

      //封装缓慢滑动到指定位置方法
private void smoothScrollTo(int desX , int desY){
    int scrollX = getScrollX();
    int deltaX = desX - scrollX;
    //指定1s内滑动到desX
    mScroller.startScroll(scrollX,0,deltaX,0,1000);
}

3.6 OverScroller

  • 官方说明:

    This class encapsulates scrolling with the ability to overshoot the bounds of a scrolling operation. This class is a drop-in replacement for Scroller in most cases.

四、自定义View的分类

  • 说明:根据有无子view可以分为2类
    • 继承自View
    • 继承自ViewGroup
  • 关于构造函数:有四种重载
    • 一个参数:一般在直接New一个View的时候调用
    • 两个参数:一般在layout文件中使用的时候会调用

4.1 继承自View

  • 第一种:实现不规则的效果,需重写onDraw方法(view中的onDraw方法为空实现)
  • 第二种:扩展某种已有的View功能(如PhotoView扩展了ImageView,支持缩放)

4.2 继承ViewGroup

  • 分类:

    • 第一种:实现自定义的布局,即除系统的布局之外,重新定义的布局(SuperSwipeRefreshLayout继承自ViewGroup)
      • 需要合适地处理ViewGroup和子view的测量和布局过程
    • 第二种:继承特定的ViewGroup(如StickyNavLayout继承自LinearLayout)
      • 用于几种View的组合,一般来说不需要处理测量和布局的过程
      • 相对于第一种方式简单

  • 说明: 如果嵌套的View可以滑动,需要合适地处理滑动冲突
